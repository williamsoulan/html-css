Transitions et Animations CSS

Les transitions et animations CSS permettent d'ajouter du mouvement et des effets visuels à vos pages web sans JavaScript, améliorant ainsi l'expérience utilisateur.

1. Transitions CSS

Les transitions permettent de modifier progressivement les propriétés CSS d'un élément, créant ainsi un effet de transition fluide entre deux états.

Propriétés de transition

  • transition-property : Propriété(s) CSS à animer
  • transition-duration : Durée de la transition
  • transition-timing-function : Courbe d'accélération
  • transition-delay : Délai avant le début de la transition
  • transition : Propriété raccourcie

Exemple de transition

.element {
    background-color: blue;
    transition-property: background-color, transform;
    transition-duration: 0.5s;
    transition-timing-function: ease;
    transition-delay: 0s;
}

/* Version raccourcie */
.element {
    background-color: blue;
    transition: background-color 0.5s ease, transform 0.5s ease;
}

.element:hover {
    background-color: red;
    transform: scale(1.2);
}

Fonctions de timing

Les fonctions de timing contrôlent la vitesse de la transition tout au long de sa durée :

linear
ease (défaut)
ease-in
ease-out
ease-in-out

2. Animations CSS

Les animations CSS permettent de créer des séquences d'animation plus complexes avec plusieurs étapes intermédiaires.

Propriétés d'animation

  • animation-name : Nom de l'animation (défini par @keyframes)
  • animation-duration : Durée de l'animation
  • animation-timing-function : Courbe d'accélération
  • animation-delay : Délai avant le début de l'animation
  • animation-iteration-count : Nombre de répétitions (nombre ou infinite)
  • animation-direction : Direction de l'animation (normal, reverse, alternate, alternate-reverse)
  • animation-fill-mode : État avant/après l'animation (none, forwards, backwards, both)
  • animation-play-state : État de lecture (running, paused)
  • animation : Propriété raccourcie

Keyframes

Les keyframes définissent les étapes intermédiaires d'une animation :

@keyframes monAnimation {
    0% {
        transform: scale(1);
        background-color: blue;
    }
    50% {
        transform: scale(1.5);
        background-color: purple;
    }
    100% {
        transform: scale(1);
        background-color: red;
    }
}

.element {
    animation-name: monAnimation;
    animation-duration: 2s;
    animation-iteration-count: infinite;
    animation-direction: alternate;
}

/* Version raccourcie */
.element {
    animation: monAnimation 2s infinite alternate;
}

3. Cas d'utilisation courants

  • Feedback visuel sur les interactions (hover, focus, active)
  • Transitions entre les états d'interface (ouverture/fermeture de menu)
  • Indicateurs de chargement
  • Animations d'entrée/sortie d'éléments
  • Effets parallaxe

Exemple de bouton interactif

.btn-interactif {
    background: linear-gradient(45deg, #3498db, #2980b9);
    transition: all 0.3s ease;
}

.btn-interactif:hover {
    transform: translateY(-5px);
    box-shadow: 0 10px 20px rgba(0, 0, 0, 0.2);
}

4. Bonnes pratiques

  • Privilégiez les transitions/animations des propriétés transform et opacity pour de meilleures performances
  • Évitez d'animer des propriétés qui déclenchent des reflows (layout, width, height, etc.)
  • Utilisez des durées raisonnables (0.2s à 0.5s pour les transitions, 1s à 3s pour les animations)
  • Respectez la préférence utilisateur prefers-reduced-motion pour l'accessibilité
  • N'abusez pas des animations qui peuvent distraire l'utilisateur
@media (prefers-reduced-motion: reduce) {
    * {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
        scroll-behavior: auto !important;
    }
}
