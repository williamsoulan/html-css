Les formulaires en HTML5

Les formulaires HTML permettent de collecter des données auprès des utilisateurs. HTML5 a introduit de nouveaux types de champs et des attributs de validation pour améliorer l'expérience utilisateur et la sécurité.

Structure de base d’un formulaire

<form action="traitement.php" method="post">
        ...
    </form>
  • action : URL où les données seront envoyées.
  • method : "get" (données dans l’URL) ou "post" (données dans le corps de la requête, plus sécurisé).

Principaux types de champs <input>

  • text : Champ de texte simple
  • email : Adresse email (validation automatique)
  • password : Mot de passe (texte masqué)
  • number : Nombre (avec min, max, step)
  • date : Date
  • tel : Numéro de téléphone
  • url : Adresse web
  • checkbox : Case à cocher
  • radio : Bouton radio
  • file : Fichier à uploader
  • range : Curseur de valeur
  • color : Sélecteur de couleur

Validation et sécurité côté client

  • required : Champ obligatoire
  • min / max : Valeur minimale et maximale (pour number, date, range...)
  • minlength / maxlength : Longueur minimale et maximale (pour text, password...)
  • pattern : Expression régulière pour valider le format (ex : numéro de téléphone, mot de passe...)

Attention : La validation HTML5 améliore la sécurité et l’expérience utilisateur, mais elle ne remplace jamais la validation côté serveur !

Exemples d’attributs de validation

<input type="text" required minlength="3" maxlength="20">
<input type="number" min="1" max="100">
<input type="email" required>
<input type="password" pattern="(?=.*[A-Z])(?=.*[0-9]).{8,}">
  • pattern : Ici, le mot de passe doit contenir au moins une majuscule, un chiffre, et faire 8 caractères minimum.

Autres éléments de formulaire

  • <textarea> : Zone de texte multiligne
  • <select> : Liste déroulante
  • <button> : Bouton (soumission, reset...)
<textarea name="message" rows="4" cols="40" required></textarea>
<select name="pays" required>
    <option value="">--Choisissez--</option>
    <option value="fr">France</option>
    <option value="be">Belgique</option>
</select>

Bonnes pratiques

  • Utiliser label pour chaque champ (accessibilité)
  • Privilégier les nouveaux types d’input pour une meilleure expérience mobile
  • Ajouter des attributs de validation pour limiter les erreurs
  • Ne jamais se fier uniquement à la validation HTML côté client
  • Protéger les données côté serveur (vérification, filtrage, protection contre les injections...)
